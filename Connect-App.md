The Connect App is the customer-facing storefront interface for the Digital Stylist platform. It provides a chat-based styling experience where customers can interact with the Ann persona—an AI stylist—to receive personalized product recommendations, itinerary-based style plans, and manage fitting-room reservations.

Technically, it is a React/TypeScript application located in `apps/connect/` that communicates with the Express Orchestration Gateway to invoke the LangGraph agent pipeline.

## 1. Multi-Day Itinerary Layout

The Connect App specializes in rendering complex, multi-day style plans. Unlike a standard chat interface, it groups AI-suggested "picks" (products) into logical day-based segments to help customers visualize an entire trip or event sequence.

### 1.1 Day Grouping Logic

The UI transforms flat lists of recommendations into a structured itinerary using `resolveConnectDayLayout` and `chunkIdsForDays`.

- `resolveConnectDayLayout`: Analyzes the state returned by the `StylistAgent` and `CatalogAgent` to determine how many days are represented in the current plan.
- `chunkIdsForDays`: Partition product IDs into arrays corresponding to specific days (e.g., Day 1: Evening Gala, Day 2: Brunch).
- `DayGroupedPicks`: A layout component that renders these chunks as distinct visual sections, often accompanied by "Stylist Notes" explaining the rationale for that specific day's selection.

### 1.2 Product Strip Rendering

Within each day, products are displayed in a horizontal or grid-based strip using `RecommendedCards`. These cards handle:

- Product imagery and metadata display.
- Direct interaction (clicking to view details).
- Feedback loops (liking/disliking) that can be fed back into the `StylistState`.

## 2. Customer Interaction & Persona

The application is themed around the Ann persona, defined as a sophisticated, helpful digital stylist. This persona is reinforced through the UI's tone, the "Stylist Notes" sections, and the specific prompts sent to the `StylistAgent`.

### 2.1 Voice Input

The Connect App integrates a `VoiceInputButton` (shared with the Clienteling app logic) that allows customers to speak their styling needs.

- Flow: Captures audio -> Sends to the `/api/v1/voice/intent` endpoint -> Receives transcribed text -> Updates the chat input.

### 2.2 Fitting-Room Reservation

A key conversion feature is the ability to reserve items for a physical fitting room.

- API Path: The app calls `POST /api/v1/fitting-room/reservations`.
- Data Flow: The reservation includes the `customer_id`, selected `product_ids`, and the target `store_id`.
- Task Creation: This action creates a task in the `stylist.appointments` table, which then appears in the Clienteling App's associate task queue.

## 3. Authentication & OTP Flow

To access personalized data (like past purchases or saved measurements), the Connect App implements a lightweight One-Time Password (OTP) flow.

1. Identity Entry: Customer enters an email or phone number.
2. OTP Challenge: The system triggers an OTP via the `CustomerAgent` or a dedicated auth utility.
3. Session Establishment: Upon successful verification, the `customer_id` is stored in the app's state and passed in the `context_metadata` of subsequent `/v1/chat` calls.

## 6. Environment Configuration

The Connect App typically runs on port `5174` in development. It relies on the Vite proxy to route `/api` and `/v1` requests to the Orchestration Gateway (port `3000`).

- `STYLIST_CORS_ORIGINS`: Must include the Connect App's URL to allow browser-side requests.
- `TRUST_PROXY`: Should be enabled if the app is deployed behind a load balancer to ensure correct protocol resolution.
